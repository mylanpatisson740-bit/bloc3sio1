== Compte-rendu des Questions
Réponses des questions de la synthèse :

=== 3.1. Différence Stored vs Reflected

*Réponse de Gedeon Razafimanjato :*
_Le Stored XSS est plus dangereux car permanent et passif. Pour piéger un admin sans soupçons, le Stored est idéal car l'injection est déjà sur le serveur, évitant l'envoi d'un lien suspect._

[green]#**Correction / Commentaire :**#
[green]#**Parfait.** Les termes "permanent" et "passif" résument exactement la menace du Stored XSS. L'absence de lien suspect est effectivement le meilleur moyen de contourner la vigilance d'un administrateur.#


=== 3.2. Le contexte d'attribut (Lab 3)

*Réponse de Gedeon Razafimanjato :*
_L'injection échoue car elle est placée dans un attribut HTML (ex: value="..."). Le navigateur traite les caractères < et > comme du texte brut ou des délimiteurs d'attributs, pas comme de nouvelles balises._

[green]#**Correction / Commentaire :**#
[green]#**Très bien.** C'est correct. Techniquement, le serveur a procédé à un **encodage en entités HTML** (transformant `<` en `&lt;`), ce qui force le navigateur à le traiter comme du "texte brut" visuel et non comme un nœud du DOM.#

=== 3.3. Syntaxe JavaScript (Lab 4)

*Réponse de Gedeon Razafimanjato :*
_Ferme la chaîne de caractères JS existante ; ; : Termine l'instruction actuelle pour en commencer une nouvelle ; // : Commente le reste du code original pour éviter les erreurs de syntaxe._

[green]#**Correction / Commentaire :**#
[green]#**Exact.** L'explication de l'évasion du contexte JavaScript (String Breakout) est irréprochable. L'objectif d'éviter les "erreurs de syntaxe" est bien identifié.#

=== 3.4. Le DOM XSS (Lab 5)

*Réponse de Gedeon Razafimanjato :*
_Le DOM XSS s'exécute côté client (navigateur) via des fragments d'URL (après le #) qui ne sont jamais envoyés au serveur, rendant l'attaque invisible pour le WAF._

[green]#**Correction / Commentaire :**#
[green]#**Brillant.** Une phrase, trois concepts clés maîtrisés : exécution côté client, rôle du fragment `#`#
[green]#(hash), et l'invisibilité pour le pare-feu applicatif.#


=== 3.5. Remédiation
