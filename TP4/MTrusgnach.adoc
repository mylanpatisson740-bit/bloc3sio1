image::labo2.png[Preuve Lab 2]

[green]#**Commentaire général :**#
[green]#La méthode technique est comprise. Vous mentionnez bien la colonne "Length" pour le username. Pour le password, quand vous dites "password code", vous faites référence au **Status Code** (302) qui change, ce qui est exact._#

=== [green]#Q3. Quel élément technique précis permet de distinguer le bon login ?#

[green]#**Correction :**#
[green]#Comme vous l'avez noté, c'est la variation de la taille de la réponse (Length) pour l'énumération du login, et le code de statut (302 Redirect) pour le mot de passe.#

=== [green]#Q4. Quelle "Bonne Pratique" permettrait d'empêcher cette attaque ?#

[green]#**Correction (Non répondu) :**#
[green]#Il faut utiliser des messages d'erreur génériques (ex: "Identifiant ou mot de passe incorrect") pour que toutes les réponses aient la même taille, empêchant ainsi l'attaquant de deviner si le compte existe.#
